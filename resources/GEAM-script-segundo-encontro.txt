Slide 3: Quando se pensa em classificação, o que vem à mente? O que "classificação" significa? 
Agrupar? Categorizar? Rotular? Definir perfis? Todas essas propriedades condizem com o conceito
de classificação, porém a definição de classificação de dados requer uma melhor fundamentação.
Classificação de dados é uma tarefa de análise onde um modelo (classificador ou classifier em
inglês) é construído para prever rótulos de classes (categorias) para os dados. Essa rotulação
pode construir categorias representadas por valores discretos onde a ordem não tem significado
particularmente relevante. A análise de dados é exemplo de previsão numérica, onde o modelo construído prevê uma
função de valor contínuo or valor ordenado ao invés de uma classe. Esse modelo é chamado de
predictor (em português, preditor, oráculo, etc.). Análise de regressão é um método estatístico
utilizado para previsão numérica, por isso que esses termos são utilizados como sinônimos por
diversos autores, embora existam outros métodos de previsão. Classificação e previsão numérica
são 2 grandes tipos de problemas de previsão.

Slide 6: O conceito de classificação acabou de ser abordado. Porém, quais etapas constituem o
processo de classificação? Classificação de dados (Data classification, em inglês) é um processo
de 2 etapas. 

Slide 7: A primeira etapa é a de aprendizagem (learning step, em inglês) responsável por construir
o modelo de classificação é treinado em função dos atributos e comportamentos extraídos da base
de dados e a etapa de classificação (classification step, em inglês) onde o modelo é usado para rotular
esses dados. A fase de aprendizagem ou treinamento é responsável por construir um cassificador para um
conjunto pré-determinado de classses de dados ou conceitos. O classificador é construído através de
um algoritmo de classificação que aprende com os tuplas de dados e seus rótulos ou classes associados. 

Slide 8: Uma tupla X é um vetor de atributos n-dimensional contendo n valores para n atributos respectivamente. Assume-se
que cada tupla X pertence a uma classe pré-definida determinada por outro atributo da base dados chamado de atributo de rótulo
de classe (valor nominal ou categórico discreto e não ordenado). Cada valor do atributo de rótulo de classe representa uma, e
apenas uma classe. Cada atributo representa um feature (propriedade, em português) de X. Sendo assim, a literatura de reconhecimento
de padrões se refere a essas tuplas como features vectors( vetores de propriedades, em português). No contexto de aprendizagem 
de máquina, tuplas de treinamento são denominadas training samples (amostras de treinamento).

Slide 9: As tuplas individuais que constituem o conjunto de treinamento são chamadas de training tuples
(tuplas de treinamento, em português). No que tange à classificação de dados, tuplas podem ser chamadas de samples, examples,
instances, data points e até objects (modelos, exemplos, instâncias, ponto de dado e objetos respectivamente em português).

Slide 10: Nesse exemplo (a) da referência bibliográfica [1], é possível visualizar o processo de classificação e compreendê-lo
de forma simples. A base de dados é consumida pelo algoritmo de classificação que por sua vez monta regras de classificação com
base no que foi capaz de aprender ao ser treinado em função das propriedades constituintes das tuplas. Essa é a etapa de aprendizagem.

Slide 11: O exemplo (b) da mesma referência permite visualizar o processo de classificação os dados de testes são utilizados para
prever a acurácia ou precisão das regras de classificação. Se a acurácia atingir um threshold(limiar, em português), as regras podem
ser aplicadas para rotular novas tuplas de dados.

Slide 12: Como a rotulação de cada tupla de treinamento já é dada, a etapa de aprendizagem é supervisionada (supervised learning, em
inglês), isto é, o algoritmo fornece a classe de cada tupla que é dada para o classificador. Por outro lado, há a aprendizagem
não supervisionada (unsupervised learning em inglês), onde a classe de cada tupla é desconhecida e o número de classes a serem
"aprendidas" pode ser ou não conhecido.

Slide 13: O que é acurácia de classificação (classification accuracy em inglês)? A acurácia preditiva do classificador é estimada
inicialmente. Se o conjunto de treinamento fosse utilizado para medir a acurácia do classificador, a estimativa seria bem otimista
e acima do limiar de aceitação pois o classificador iriam sobreajustar os dados. Como ocorre esse sobreajuste? É simples, o classificador
incorpora anomalias dos dados de treinamento que não fazem parte do dataset como um todo. É possível perceber que isso levaria
a resultados indesejados.

Slide 14: Assim o ideal é utilizar um conjunto de testes (test set, em inglês) ao invés de um training set, pois as tuplas de teste
são independentes no sentido que não influenciaram na construção do classificador. Por fim, A acurácia de um classificador é a
porcentagem de tuplas de teste que foram classificados corretamente de acordo com suas classes. Se a acurácia atingir um determinado
limiar de aceitação, então o classificador poderá ser utilizado para classificar novas tuplas de dados cujos rótulos são desconhecidos.
